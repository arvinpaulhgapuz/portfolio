# Cisco SOC Essentials – Investigating with Splunk

## Overview

This course introduces learners to the core responsibilities of a Security Operations Center (SOC) analyst, with a focus on using **Splunk** as the primary tool for security monitoring, log analysis, and incident investigation. Built around Cisco's SOC training framework, the course blends foundational security concepts with hands-on investigative practice, preparing learners to detect, analyze, and respond to security events in a real-world SOC environment.

By the end of this course, learners will be able to navigate Splunk confidently, build effective searches, interpret security alerts, and carry out structured incident investigations from initial detection through to resolution.

---

## Who This Course Is For

- Aspiring SOC analysts (Tier 1 / Tier 2)
- IT and network professionals transitioning into cybersecurity
- Students pursuing security certifications (e.g., CompTIA Security+, Cisco CyberOps)
- Anyone seeking practical, tool-based experience in security monitoring

## Prerequisites

- Basic understanding of networking concepts (TCP/IP, DNS, HTTP)
- Familiarity with operating systems (Windows/Linux fundamentals)
- General awareness of common cybersecurity terminology is helpful but not required

---

## Learning Objectives

By completing this course, learners will be able to:

1. Explain the role and structure of a Security Operations Center (SOC)
2. Describe the SOC analyst workflow and incident response lifecycle
3. Navigate the Splunk interface and understand its core architecture
4. Build and refine Splunk Search Processing Language (SPL) queries
5. Create and interpret dashboards, reports, and visualizations
6. Investigate security alerts using log correlation and pivoting techniques
7. Identify indicators of compromise (IOCs) within log data
8. Apply structured investigation methodology to sample security incidents
9. Document findings and escalate incidents appropriately

---

## Course Outline

### Module 1: Introduction to SOC Operations
- What is a Security Operations Center?
- SOC tiers and analyst roles (Tier 1, Tier 2, Tier 3)
- The incident response lifecycle (NIST/Cisco frameworks)
- Common SOC tools and data sources (SIEM, IDS/IPS, firewalls, endpoint logs)

### Module 2: Splunk Fundamentals
- Splunk architecture overview (forwarders, indexers, search heads)
- Navigating the Splunk Web interface
- Understanding indexes, sourcetypes, and fields
- Ingesting and exploring sample data sets

### Module 3: Search Processing Language (SPL) Basics
- Writing basic search queries
- Filtering, sorting, and field extraction
- Using `stats`, `table`, `eval`, and `where` commands
- Time range modifiers and search optimization

### Module 4: Dashboards, Reports, and Alerts
- Building visualizations and dashboards
- Creating scheduled reports
- Configuring and tuning alerts
- Reducing alert fatigue through effective thresholds

### Module 5: Log Analysis and Correlation
- Working with firewall, proxy, DNS, and endpoint logs
- Correlating events across multiple data sources
- Identifying patterns indicative of malicious activity
- Introduction to MITRE ATT&CK mapping in Splunk

### Module 6: Investigating Security Incidents
- Triage methodology: validating and prioritizing alerts
- Pivoting from an alert to root cause
- Identifying Indicators of Compromise (IOCs)
- Case study: investigating a simulated phishing/malware incident in Splunk

### Module 7: Incident Documentation and Escalation
- Writing clear, actionable incident reports
- Escalation procedures and handoff to Tier 2/3
- Post-incident review and lessons learned

### Module 8: Capstone Investigation Lab
- End-to-end simulated investigation using Splunk
- Learners analyze logs, identify the incident, and produce a written report
- Peer or instructor review of findings
